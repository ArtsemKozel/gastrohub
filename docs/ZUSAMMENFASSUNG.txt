 WAS IST GASTROHUB?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GastroHub ist eine Neuentwicklung auf Basis von PlanIt. PlanIt war eine
monolithische Single-Page-App (eine große admin.js + employee.js). GastroHub
zerlegt diese Logik in eine modulare Ordnerstruktur mit getrennten JS-Dateien
pro Feature-Bereich und drei klar getrennten Nutzergruppen.

Diese Regeln gelten für alle Aufträge an Claude Code in diesem Projekt und
sollen eine konsistente, nachvollziehbare Zusammenarbeit sicherstellen.

Aufträge werden als zusammenhängender Fließtext auf Deutsch formuliert —
keine Aufzählungslisten, keine Codeblöcke im Auftragstext. Der Auftragstext
nennt immer den Dateinamen und den Funktionsnamen, der geändert werden soll,
damit Claude Code ohne Rückfragen an die richtige Stelle navigieren kann.

Pro Auftrag wird genau eine inhaltliche Änderung beschrieben. Mehrere
unabhängige Änderungen werden als separate Aufträge gestellt, nicht
zusammengefasst, um Verwechslungen beim Commit-Text zu vermeiden und die
Git-Historie übersichtlich zu halten.

Mobile und Desktop werden immer gleichzeitig berücksichtigt. Wann immer eine
UI-Änderung in einer Ansicht gemacht wird, muss geprüft werden ob dieselbe
Änderung auch für die andere Bildschirmgröße passt oder angepasst werden muss.

Nach jedem abgeschlossenen Feature oder jeder Änderung wird der Stand sofort
committed und zu GitHub gepusht. Es wird nicht auf mehrere fertige Aufgaben
gewartet — jede Änderung erhält ihren eigenen Commit mit aussagekräftigem
Commit-Text auf Deutsch.
